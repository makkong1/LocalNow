# 백엔드 리팩터링 요약 (공식 API · 보안 정렬)

## 배경

모바일이 백엔드를 직접 호출하면 BFF 뒤에 있던 **조회/실시간 구독 권한** 이 그대로 노출된다. 문서(`docs/API_CONVENTIONS.md`, `phases/0-mvp/backend-gap-analysis.md`)에 맞춰 **단일 Spring API** 를 공식 계약으로 두고, HTTP/WebSocket 모두 **당사자·역할** 기준을 명확히 했다.

## 변경 요지

1. **요청**
   - `GET /requests/open` — `HelpRequestStatus.OPEN` 만, GUIDE 전용, cursor 페이징.
   - `GET /requests/{id}` — TRAVELER 는 본인 요청만, GUIDE 는 OPEN 이거나 해당 요청에 offer 가 있는 경우만.
2. **매칭**
   - `GET /requests/{id}/offers` — TRAVELER 는 해당 요청의 소유자만, GUIDE 는 OPEN 이거나 offer 로 연관된 경우만.
3. **결제**
   - `GET /payments/{requestId}` — 결제 intent 의 **payer(여행자)** 또는 **payee(가이드)** 만 조회.
4. **STOMP**
   - `/topic/users/{userId}`, `/topic/guides/{guideId}`, `/topic/requests/{requestId}` 구독 시 JWT 와 path id·요청 권한 검증. 알림 **topic 만으로는 비밀 유지가 되지 않으므로** 인터셉터에서 막는다.
   - `/ws-native` — React Native·`@stomp/stompjs` 가 쓰는 **네이티브 WebSocket** 엔드포인트. 브라우저용 SockJS 는 기존 `/ws` 유지.
5. **가이드 duty**
   - `GuideController` 가 `Principal#getName` 에 의존하던 부분을 제거하고, 다른 컨트롤러와 동일하게 `Authentication` principal (`Long` userId) 사용.
   - `@PreAuthorize` 가 실제로 동작하도록 `@EnableMethodSecurity` 활성화.

## 검증

`cd backend && ./gradlew check` (단위/통합 테스트 포함) 통과를 기준으로 한다.

## 웹/모바일 소비

- **웹** — Route Handler 는 동일 endpoint 를 Bearer 로 proxy; 계약은 이 백엔드와 1:1.
- **모바일** — 동일 URL; WebSocket 은 `EXPO_PUBLIC_*` 로 노출되는 베이스 URL 만 사용하고, STOMP 는 `/ws-native` 로 연결.
