## 권장 실행 순서 (요약)

1. **P0-인프라**: Testcontainers IT가 실제로 돌게 환경 정비 → 동시성·플로우 검증 복구
2. **P0-데이터/실시간**: Redis 온듀티 TTL·정리 → 잘못된 가이드 알림 방지
3. **P0-구조**: `match` ↔ `chat` 결합 완화(이벤트/포트)
4. **P1-보안·API**: 로그인/가입 레이트 리밋, CORS 운영 주입, (필요 시) Refresh 토큰
5. **P1-웹·모바일 UX**: 웹 401 처리·e2e, 폴링 정리, 모바일 API 오류/401 처리
6. **모바일·스토어(Phase 4)**: 앱 안정화(MOB-P1-*) 후 OAuth 콜백·HTTPS·권한 문구·EAS·프라이버시 라벨
7. **P2**: OpenAPI 설명 보강, API `/v1`, 관측성·프로파일·아웃박스 등

아래 표는 **같은 단계 안에서도 P0 먼저** 적용하면 됨. **앱스토어 목표 시** 백엔드 P0와 병행하되, 사용자 경험은 4번→6번 순을 우선해도 됨.
