# LocalNow — 이력서·경력기술서용 요약

아래 문장은 그대로 복사해 경력기술서·이력서의 **프로젝트** 블록에 붙여 넣을 수 있도록 작성했다. 톤만 본인 스타일에 맞게 다듬으면 된다.

---

## 한 줄 요약

> 여행자와 현지 가이드를 **위치·실시간** 기준으로 매칭하고, **WebSocket 채팅·결제(Mock)·리뷰**까지 이어지는 풀스택 MVP를 **Spring Boot 백엔드 + Next.js 웹 + React Native(Expo) 모바일**로 구현한 개인 포트폴리오 프로젝트.

---

## 역할 (예시 문장)

- **풀스택(1인)**: 백엔드 API 설계·구현, 웹 데모 클라이언트, 모바일 앱 클라이언트, 인프라(docker-compose)·문서(PRD/ADR/API 규약) 정리.
- **백엔드**: 도메인 모델링, REST API, JWT·OAuth2 연동 흐름, Redis GEO·분산락, RabbitMQ 이벤트, STOMP WebSocket, Flyway 마이그레이션, Testcontainers 기반 통합 테스트.
- **프론트**: Next.js(App Router) 웹 데모, React Native(Expo) 모바일 앱 — TanStack Query, 실시간 STOMP 클라이언트, Secure Store 기반 토큰 저장 등.

(실제 본인 기여에 맞게 “백엔드 위주” / “모바일 추가 개발” 등으로 줄여도 됨.)

---

## 기술 스택 (이력서 스킬 라인에 넣기 좋은 형태)

**Backend:** Java 17, Spring Boot 3.3, Spring Security, JPA/Hibernate, MySQL, Flyway, Redis(GEO·캐시·분산락), RabbitMQ, WebSocket(STOMP), JWT, Gradle, JUnit 5, Testcontainers  

**Web:** Next.js 15, React 19, TypeScript, TanStack Query, Tailwind CSS, STOMP/SockJS, Leaflet  

**Mobile:** React Native 0.76, Expo SDK 52, TypeScript(strict), NativeWind, TanStack Query, STOMP(네이티브 WebSocket), React Navigation, expo-secure-store, expo-location, react-native-maps  

**DevOps·기타:** Docker Compose(MySQL/Redis/RabbitMQ), API/아키텍처 문서화(PRD, ADR)

---

## 구현 하이라이트 (불릿 — 성과 문장으로 가공)

- **실시간 매칭·채팅**: 도움 요청·가이드 오퍼·확정 이후 동일 세션에서 STOMP 기반 1:1 채팅; 웹·모바일 클라이언트 모두 동일 백엔드 계약을 소비.
- **위치 기반**: Redis GEO로 주변 가이드/요청 탐색; 모바일에서는 GPS·지도(OSM)와 연계한 요청 생성·표시.
- **동시성**: 매칭 확정 등 경쟁 상태에 **분산락·낙관적 락**을 적용해 중복 확정·충돌 시 명확한 에러 응답.
- **비동기 이벤트**: RabbitMQ로 도메인 이벤트 발행·소비, 알림·연동 지점 분리.
- **결제 도메인**: 실 PG 없이 **Mock PG**로 상태 전이·수수료·정산 흐름을 재현(실연동은 범위 외로 명시).
- **품질**: 백엔드 `gradlew check`, 모바일 Jest/ESLint, 웹 빌드·린트; 계층형 아키텍처 규칙(컨트롤러는 매핑만, 비즈니스는 서비스) 유지.

---

## 숫자가 없을 때 쓰는 안전한 표현

- “MVP 범위 내 **엔드투엔드** 시나리오(요청 → 매칭 → 채팅 → 결제(Mock) → 리뷰) 구현”
- “**Testcontainers**로 외부 의존성을 띄운 통합 테스트”
- “**문서화된 API 규약·ADR**로 설계 결정 추적”

(트래픽·DAU 등 운영 지표가 없으면 억지 숫자는 넣지 않는 것이 좋다.)

## 면접 짧은 답변용 (30초)

“LocalNow는 여행 중 즉시 도움이 필요할 때 현지 가이드와 매칭하는 서비스예요. 백엔드는 Spring Boot로 REST·JWT·OAuth, Redis로 위치 검색이랑 매칭 확정 때 동시성 제어, RabbitMQ로 이벤트, WebSocket으로 채팅까지 넣었고요. 웹은 Next.js 데모, 앱은 Expo로 같은 API를 씁니다. 결제는 실 PG 대신 Mock으로 상태 머신만 검증했습니다.”
